# 变更提案：全量地图路线规划

## 背景

当前 MAP 状态虽然能告诉 agent 下一步可以走哪里，但如果只展示当前位置和下一步选项，Route Agent 只能做贪心选择，无法判断“精英前有没有火堆”“商店在路线前段还是后段”“是否连续多层小怪”等长期路线信息。

## 目标

- C# Mod 输出稳定的地图节点身份和图结构字段。
- Runtime 从当前位置开始枚举所有剩余路线，而不是从起点重新统计。
- Route Agent 在 prompt 中看到每条路线的未来顺序、未来总览和对应的下一步 legal action。

## 非目标

- 不让代码替大模型判断路线好坏。
- 不实现复杂路线打分器。
- 不把已经走过的节点计入未来路线统计。
